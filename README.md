# Shop Mini App — Phase 3

แอพคิดต้นทุน + ขายหน้าร้าน + สรุปกำไร สำหรับร้านค้าขนาดเล็ก
HTML + Pure CSS + Vanilla JS — ติดตั้งบนมือถือได้ ใช้งาน offline ได้

## โครงสร้าง

| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | หน้าขาย (POS) |
| `menu.html` | สร้างเมนู + ผูกสูตรวัตถุดิบ |
| `ingredients.html` | จัดการวัตถุดิบ + บันทึกการซื้อ |
| `summary.html` | สรุปยอดขาย / ต้นทุน / กำไร |
| `settings.html` | ชื่อร้าน + Export/Import JSON |

## ลำดับการใช้งานครั้งแรก

1. แท็บ **วัตถุดิบ** → เพิ่มวัตถุดิบ (กรอก "ซื้อมาเท่าไหร่ ได้กี่หน่วย" ระบบคำนวณต้นทุนต่อหน่วยให้)
2. แท็บ **เมนู** → สร้างเมนู เลือกวัตถุดิบเข้าสูตร เห็นต้นทุน/กำไรทันที
3. แท็บ **ขาย** → กดเมนู → ยืนยันการขาย (ระบบตัดสต็อกอัตโนมัติ)
4. แท็บ **สรุป** → ดูยอดขาย กำไรขั้นต้น กำไรเงินสด
5. แท็บ **ตั้งค่า** → Export backup ทุกสิ้นวัน

## Deploy ขึ้น GitHub Pages

1. สร้าง repo ใหม่ (เช่น `shop-app`) แล้ว push ไฟล์ทั้งหมดขึ้นไป
   ```
   git init
   git add .
   git commit -m "Phase 1"
   git branch -M main
   git remote add origin https://github.com/<username>/shop-app.git
   git push -u origin main
   ```
2. ไปที่ repo → **Settings → Pages** → Source: `Deploy from a branch` → Branch: `main` / `(root)` → Save
3. รอ ~1 นาที เปิดได้ที่ `https://<username>.github.io/shop-app/`

ทุก path ในโปรเจกต์เป็น relative (`./`) อยู่แล้ว รองรับการรันใน subfolder ของ GitHub Pages

## ข้อมูลเก็บที่ไหน

เก็บใน `localStorage` ของ browser เครื่องนั้นๆ — ผูกกับเครื่อง/browser เดียว
ใช้ปุ่ม **Export** ในแท็บตั้งค่าเพื่อ backup เป็นไฟล์ `.json` และ **Import** เพื่อย้ายข้อมูลข้ามเครื่อง

## ติดตั้งเป็นแอพบนมือถือ (PWA)

ต้อง deploy ขึ้น GitHub Pages (หรือ host ที่เป็น HTTPS) ก่อน — Service Worker ไม่ทำงานกับการเปิดไฟล์ตรงๆ (`file://`)

**iPhone (Safari):** เปิดเว็บ → ปุ่ม Share → **Add to Home Screen** → ได้ไอคอนส้มบนหน้าจอ เปิดแบบเต็มจอไม่มีแถบ browser
**Android (Chrome):** เปิดเว็บ → เมนู ⋮ → **Add to Home screen / Install app**

หลังเปิดครั้งแรก แอพจะ cache ไฟล์ทั้งหมดไว้ — ครั้งถัดไป **เปิดได้แม้ไม่มีเน็ต**

ข้อควรรู้บน iOS: ข้อมูล localStorage ของแอพที่ติดตั้งจะแยกจาก Safari ปกติ และควร Export backup สม่ำเสมอ

## เวลาแก้โค้ดแล้ว deploy ใหม่

แก้เลขเวอร์ชันใน `sw.js` ทุกครั้ง ไม่งั้นเครื่องลูกค้าจะยังใช้ไฟล์เก่าจาก cache:
```js
const CACHE_VERSION = 'shop-app-v2';   // v1 → v2 → v3 ...
```

## ฟีเจอร์ Phase 3

- **จุดเตือนใกล้หมด** — ตั้งจุดเตือนต่อวัตถุดิบได้ (เช่น เหลือต่ำกว่า 100g ให้เตือน) → badge "ใกล้หมด" + จุดแดงที่แท็บวัตถุดิบ + เตือนตอนขาย
- **ยกเลิกออเดอร์** — หน้าสรุป แตะออเดอร์ล่าสุด → ยืนยัน → ระบบคืนสต็อกตาม snapshot ที่บันทึกตอนขาย (แม่นยำแม้สูตรถูกแก้ภายหลัง)
- **กราฟยอดขาย 7 วัน** — SVG ล้วน ไม่ใช้ library แท่งวันนี้สีส้มเข้ม
- **เตือน backup** — ถ้ามียอดขายแต่ไม่ได้ Export เกิน 3 วัน จะมีจุดแดงที่แท็บตั้งค่า + สถานะ backup ล่าสุดในหน้าตั้งค่า
